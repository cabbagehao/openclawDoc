# 스크립트

`scripts/` 디렉터리에는 로컬 워크플로와 운영 작업을 돕는 보조 스크립트가 들어 있습니다.
작업이 특정 스크립트와 명확히 연결되어 있을 때 사용하고, 그렇지 않으면 CLI를 우선하세요.

## 규칙

- 문서나 릴리스 체크리스트에서 참조하지 않는 한 스크립트는 **선택 사항**입니다.
- 가능한 경우 CLI 표면을 우선하세요. 예를 들어 인증 모니터링은 `openclaw models status --check`를 사용합니다.
- 스크립트는 호스트별 차이가 있을 수 있다고 가정하고, 새 머신에서 실행하기 전에 먼저 읽어보세요.

## 스크립트를 추가할 때

- 스크립트는 한 가지 목적에 집중시키고 문서화하세요.
- 관련 문서에 짧은 항목을 추가하세요. 없다면 새로 만드세요.
