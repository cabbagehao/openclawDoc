# 구성 예시

아래 예시는 현재 config schema에 맞춰져 있습니다. 전체 레퍼런스와 필드별 설명은 [Configuration](/gateway/configuration)을 참고하세요.

## 빠른 시작

### 절대 최소 구성

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

이를 `~/.openclaw/openclaw.json`에 저장하면, 해당 번호에서 봇에게 DM을 보낼 수 있습니다.

### 권장 시작 구성

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-5" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## 일반 패턴

### 멀티 플랫폼 구성

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### 보안 DM 모드(공유 inbox / 다중 사용자 DM)

봇에 DM을 보낼 수 있는 사람이 둘 이상이라면(`allowFrom`에 여러 항목이 있거나, 여러 사람에 대한 pairing 승인, 또는 `dmPolicy: "open"`인 경우), 다른 발신자의 DM이 기본적으로 하나의 컨텍스트를 공유하지 않도록 **보안 DM 모드**를 활성화하세요.

```json5
{
  // Secure DM mode (recommended for multi-user or sensitive DM agents)
  session: { dmScope: "per-channel-peer" },

  channels: {
    // Example: WhatsApp multi-user inbox
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // Example: Discord multi-user inbox
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```

Discord/Slack/Google Chat/MS Teams/Mattermost/IRC에서는 발신자 인증이 기본적으로 ID 우선입니다.
그 위험을 명시적으로 감수할 때만 각 채널의 `dangerouslyAllowNameMatching: true`로 직접 변경 가능한 이름/이메일/nick 매칭을 활성화하세요.

### OAuth + API key failover

```json5
{
  auth: {
    profiles: {
      "anthropic:subscription": {
        provider: "anthropic",
        mode: "oauth",
        email: "me@example.com",
      },
      "anthropic:api": {
        provider: "anthropic",
        mode: "api_key",
      },
    },
    order: {
      anthropic: ["anthropic:subscription", "anthropic:api"],
    },
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: {
      primary: "anthropic/claude-sonnet-4-5",
      fallbacks: ["anthropic/claude-opus-4-6"],
    },
  },
}
```

### Anthropic setup-token + API key, MiniMax fallback

<Warning>
과거에 일부 사용자의 경우 Claude Code 외부에서 Anthropic setup-token 사용이 제한된 적이 있습니다. 이를 사용자 선택에 따른 위험으로 취급하고, subscription auth에 의존하기 전에 현재 Anthropic 약관을 확인하세요.
</Warning>

```json5
{
  auth: {
    profiles: {
      "anthropic:subscription": {
        provider: "anthropic",
        mode: "oauth",
        email: "user@example.com",
      },
      "anthropic:api": {
        provider: "anthropic",
        mode: "api_key",
      },
    },
    order: {
      anthropic: ["anthropic:subscription", "anthropic:api"],
    },
  },
  models: {
    providers: {
      minimax: {
        baseUrl: "https://api.minimax.io/anthropic",
        api: "anthropic-messages",
        apiKey: "${MINIMAX_API_KEY}",
      },
    },
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: {
      primary: "anthropic/claude-opus-4-6",
      fallbacks: ["minimax/MiniMax-M2.5"],
    },
  },
}
```

## 팁

- `dmPolicy: "open"`을 설정하면, 일치하는 `allowFrom` 목록에 `"*"`가 포함되어 있어야 합니다.
- Provider ID 형식은 서로 다릅니다(전화번호, 사용자 ID, 채널 ID). 형식은 provider 문서에서 확인하세요.
- 나중에 추가할 수 있는 선택적 섹션: `web`, `browser`, `ui`, `discovery`, `canvasHost`, `talk`, `signal`, `imessage`.
- 더 깊은 설정 설명은 [Providers](/providers)와 [Troubleshooting](/gateway/troubleshooting)을 참고하세요.
