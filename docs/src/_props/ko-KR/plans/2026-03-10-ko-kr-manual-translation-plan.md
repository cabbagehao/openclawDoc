# ko-KR 수동 번역 계획

> **Codex용:** 이 워크스페이스의 영어 원문 문서를 직접 번역합니다. 내용 생성에 Go 번역 파이프라인이나 `claude` CLI를 사용하지 않습니다.

**목표:** `docs/ko-KR/**`를 확장해 한국어 문서 커버리지가 영어 문서 트리와 자연스럽고 일관된 한국어 품질로 맞춰지도록 합니다.

**아키텍처:** 영어 문서 트리를 기준 진실의 원천으로 사용합니다. 배치 단위로 페이지를 직접 번역하면서 frontmatter 키, 링크, 코드 펜스, Mintlify 컴포넌트, 파일 경로, 제품명을 보존하고, 서술형 문장은 자연스러운 한국어로 다시 씁니다. 핵심 용어는 `docs/.i18n/glossary.ko-KR.json`에 중앙 관리하고, 각 배치 후 커버리지를 검증합니다.

**기술 스택:** Markdown/MDX 문서, Mintlify `docs.json`, 로컬 Node 스크립트, Codex 기반 수동 번역, 선택적 커버리지 및 링크 검사.

### 작업 1: 한국어 번역 규칙 정립

**파일:**

* 수정: `docs/.i18n/glossary.ko-KR.json`
* 참조: `docs/ADDING_NEW_LANGUAGE.md`
* 참조: `docs/docs.json`

**단계 1:** 핵심 제품 명사와 UI 라벨에 대한 안정적인 한국어 용어를 정의합니다.

**단계 2:** 제품명, CLI 명령, 환경 변수, 코드, URL, 설정 키는 영어 원형을 유지합니다.

**단계 3:** 영어 문서 트리만 번역 원본으로 사용하고, 일본어 로케일 내용은 참조하지 않습니다.

### 작업 2: 섹션 단위 배치로 문서 번역

**파일:**

* 생성: `docs/ko-KR/**`
* 참조: `docs/**/*.md`
* 참조: `docs/**/*.mdx`

**단계 1:** 먼저 유입이 많은 진입 페이지와 짧은 루트 문서부터 처리합니다.

**단계 2:** `start`, `web`, `channels`, `cli`, `concepts`처럼 서로 겹치지 않는 섹션을 한 번에 하나씩 번역합니다.

**단계 3:** 보이는 텍스트만 한국어로 번역하고, frontmatter 구조, 컴포넌트 태그, 이미지, 링크 대상은 유지합니다.

**단계 4:** 한국어가 직역 문장처럼 보이지 않도록 문맥을 보면서 문장을 다듬습니다.

### 작업 3: 충분한 페이지 커버리지 이후 내비게이션 동기화

**파일:**

* 수정: `docs/docs.json`
* 참조: `docs/.i18n/navigation.ko-KR.json`

**단계 1:** 커버리지가 늘어나는 흐름에 맞춰 사이드바 라벨을 번역합니다.

**단계 2:** 참조되는 페이지 세트가 실제로 존재할 때만 한국어 내비게이션을 동기화합니다.

**단계 3:** 탭, 그룹, 페이지 순서는 영어와 동일하게 유지합니다.

### 작업 4: 각 배치 후 검증

**파일:**

* 검증: `docs/ko-KR/**`
* 검증: `docs/docs.json`

**단계 1:** 각 배치 후 `node scripts/check-coverage.mjs --lang ko-KR --docs docs`를 실행합니다.

**단계 2:** 가능할 때마다 변경한 파일에 대해 포맷 및 링크 검사를 실행합니다.

**단계 3:** 번역 누락, 용어 흔들림, 내부 링크 문제를 수정한 뒤 다음 배치로 넘어갑니다.
