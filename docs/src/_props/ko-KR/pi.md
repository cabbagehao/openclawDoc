# Pi 통합 아키텍처

이 문서는 OpenClaw가 AI 에이전트 기능을 위해 [pi-coding-agent](https://github.com/badlogic/pi-mono/tree/main/packages/coding-agent) 및 관련 패키지(`pi-ai`, `pi-agent-core`, `pi-tui`)와 통합된 아키텍처를 설명함.

## 개요

OpenClaw는 Pi SDK를 활용하여 AI 코딩 에이전트를 메시징 게이트웨이 아키텍처에 직접 통합함. 별도의 하위 프로세스나 RPC 모드 대신, `createAgentSession()`을 호출하여 Pi의 `AgentSession`을 인라인으로 생성 및 관리함. 이러한 내장형(Embedded) 접근 방식의 장점은 다음과 같음:

* 세션 생명주기 및 이벤트 처리에 대한 완전한 제어권 확보
* 채널 및 컨텍스트별 맞춤형 도구(메시징, 샌드박스 등) 주입 용이
* 유연한 시스템 프롬프트 커스터마이징 가능
* 브랜칭(Branching) 및 압축(Compaction)을 지원하는 정교한 세션 영속성 관리
* 다중 계정 기반 인증 프로필 순환 및 장애 조치(Failover) 지원
* 공급자 제약 없는 동적인 모델 전환 가능

## 주요 패키지 의존성

```json
{
  "@mariozechner/pi-agent-core": "0.49.3",
  "@mariozechner/pi-ai": "0.49.3",
  "@mariozechner/pi-coding-agent": "0.49.3",
  "@mariozechner/pi-tui": "0.49.3"
}
```

| 패키지 명             | 용도                                                |
| ----------------- | ------------------------------------------------- |
| `pi-ai`           | 핵심 LLM 추상화: `Model`, 스트리밍 API, 메시지 타입 정의 및 공급자 연동 |
| `pi-agent-core`   | 에이전트 실행 루프, 도구 실행 엔진, `AgentMessage` 구조 정의        |
| `pi-coding-agent` | 고수준 SDK: 세션 관리자, 인증 저장소, 모델 레지스트리 및 내장 코딩 도구 제공   |
| `pi-tui`          | 터미널 UI 컴포넌트 라이브러리 (OpenClaw 로컬 TUI 모드에서 사용됨)      |

## 핵심 통합 프로세스

### 1. 내장형 에이전트 실행

주요 진입점은 `pi-embedded-runner/run.ts` 파일의 `runEmbeddedPiAgent()` 함수임. 이 함수는 세션 ID, 워크스페이스 경로, 설정 정보 등을 입력받아 에이전트 런타임을 구성함.

### 2. 세션 생성 및 초기화

`runEmbeddedAttempt()` 단계에서 Pi SDK를 호출하여 `AgentSession`을 인스턴스화함. 이때 로컬 리소스 로더, 인증 저장소, 모델 레지스트리 등이 함께 구성되며, 필요한 시스템 프롬프트 오버라이드가 적용됨.

### 3. 이벤트 구독 시스템

`subscribeEmbeddedPiSession()`을 통해 Pi 에이전트의 내부 이벤트(메시지 상태 업데이트, 도구 실행 시작/종료, 에이전트 상태 변화 등)를 수신하고 대처함. 이를 통해 실시간 스트리밍 답변과 생각(Reasoning) 과정을 사용자에게 전달함.

### 4. 프롬프트 처리 및 루프 실행

설정이 완료된 세션에 사용자의 입력을 전달하면, SDK는 LLM과의 통신, 필요한 도구 호출 실행, 응답 조립으로 이어지는 전체 에이전트 루프를 자율적으로 수행함.

## 도구(Tool) 및 확장 시스템

### 도구 파이프라인

OpenClaw는 Pi의 기본 도구를 기반으로 독자적인 도구 체인을 구축함:

1. **커스텀 도구 대체**: 기본 Bash 도구를 `exec`/`process` 도구로 교체하고, 샌드박스 환경에 최적화된 파일 읽기/쓰기 도구를 주입함.
2. **OpenClaw 전용 도구**: 메시징, 브라우저 제어, 캔버스 조작, 세션 관리 등 70개 이상의 커스텀 도구 제공.
3. **정책 기반 필터링**: 보안 정책에 따라 각 에이전트 및 샌드박스 환경에서 사용 가능한 도구를 동적으로 제한함.

### Pi 확장(Extensions) 로드

특수한 동작 제어를 위해 다음과 같은 Pi 확장을 로드함:

* **압축 보호장치(Compaction Safeguard)**: 적응형 토큰 관리 및 압축 중 실패 방지 로직 제공.
* **컨텍스트 가지치기(Context Pruning)**: 캐시 수명(TTL) 기반의 효율적인 컨텍스트 관리 기법 적용.

## 세션 관리 및 영속성

### 세션 데이터 구조

세션은 트리 형태의 JSONL 파일로 저장되며, Pi의 `SessionManager`가 이를 총괄함. OpenClaw는 이 세션 파일 파싱 성능을 위해 별도의 인스턴스 캐싱 메커니즘을 적용함.

### 대화 이력 및 압축 관리

채널 특성에 따라 대화 이력 길이를 조절하며, 컨텍스트 용량 임계값 도달 시 자동으로 압축을 수행하여 대화의 연속성을 유지함.

## 인증 및 모델 해석

### 다중 프로필 시스템

공급자별로 여러 API 키를 보유할 수 있는 인증 프로필 시스템을 지원함. 오류 발생 시 자동으로 다음 프로필로 순환하거나 다른 모델로의 장애 조치(Failover)를 수행함.

### 모델 해석 로직

Pi의 `ModelRegistry`를 활용하여 요청된 공급자와 모델 ID를 해석하고, 해당 런타임에 최적화된 인증 토큰을 동적으로 주입함.

| 인증 체계    | 단일 자격 증명 사용      | 다중 프로필 순환 및 자동 장애 조치           |
