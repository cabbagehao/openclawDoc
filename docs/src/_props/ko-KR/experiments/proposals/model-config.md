# 모델 설정 (탐색)

이 문서는 향후 모델 설정에 대한 **아이디어** 를 정리한 것이다. 배포용 사양은 아니다. 현재 동작은 다음 문서를 참고한다.

* [Models](/concepts/models)
* [Model failover](/concepts/model-failover)
* [OAuth + profiles](/concepts/oauth)

## 동기

운영자는 다음을 원한다.

* 공급자별 다중 인증 프로필(개인용 vs 업무용)
* 예측 가능한 폴백과 함께 단순한 `/model` 선택
* 텍스트 모델과 이미지 처리 가능 모델 간의 명확한 분리

## 가능한 방향(고수준)

* 모델 선택은 단순하게 유지한다: 선택 형식은 `provider/model`, 필요하면 별칭을 추가한다.
* 공급자는 명시적 순서를 가진 여러 인증 프로필을 가질 수 있게 한다.
* 모든 세션이 일관되게 폴백되도록 전역 폴백 목록을 사용한다.
* 이미지 라우팅은 명시적으로 설정된 경우에만 재정의한다.

## 열린 질문

* 프로필 회전은 공급자 단위여야 할까, 모델 단위여야 할까?
* UI 는 세션별 프로필 선택을 어떻게 노출해야 할까?
* 레거시 설정 키에서 마이그레이션하는 가장 안전한 경로는 무엇일까?
