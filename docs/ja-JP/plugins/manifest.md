# プラグインマニフェスト (openclaw.plugin.json)

すべてのプラグインは **プラグイン ルート**に `openclaw.plugin.json` ファイルを**提供する必要があります**。
OpenClaw は、このマニフェストを使用して **プラグインを実行せずに構成を検証します
コード**。不足しているマニフェストまたは無効なマニフェストはプラグイン エラーとして扱われ、ブロックされます。
構成の検証。

完全なプラグイン システム ガイドを参照してください: [プラグイン](/tools/plugin)。

## 必須フィールド

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

必要なキー:

- `id` (文字列): 正規のプラグイン ID。
- `configSchema` (オブジェクト): プラグイン構成の JSON スキーマ (インライン)。

オプションのキー:

- `kind` (文字列): プラグインの種類 (例: `"memory"`、`"context-engine"`)。
- `channels` (配列): このプラグインによって登録されたチャンネル ID (例: `["matrix"]`)。
- `providers` (配列): このプラグインによって登録されたプロバイダー ID。
- `skills` (配列): ロードするスキル ディレクトリ (プラグイン ルートに相対)。
- `name` (文字列): プラグインの表示名。
- `description` (文字列): プラグインの短い概要。
- `uiHints` (オブジェクト): UI レンダリング用の構成フィールド ラベル/プレースホルダー/機密フラグ。
- `version` (文字列): プラグインのバージョン (情報)。

## JSON スキーマの要件

- **すべてのプラグインは、構成を受け入れない場合でも、JSON スキーマを同梱する必要があります**。
- 空のスキーマは許容されます (`{ "type": "object", "additionalProperties": false }` など)。
- スキーマは実行時ではなく、構成の読み取り/書き込み時に検証されます。## 検証動作

- チャネル ID が宣言されていない限り、不明な `channels.*` キーは **エラー**
  プラグインマニフェスト。
- `plugins.entries.<id>`、`plugins.allow`、`plugins.deny`、および `plugins.slots.*`
  **検出可能な**プラグイン ID を参照する必要があります。不明な ID は **エラー** です。
- プラグインがインストールされているが、マニフェストまたはスキーマが壊れているか欠落している場合、
  検証が失敗し、Doctor がプラグイン エラーを報告します。
- プラグイン構成は存在するが、プラグインが **無効になっている**場合、構成は保持され、
  Doctor + のログに **警告** が表示されます。

## 注意事項

- マニフェストは、ローカル ファイルシステムのロードを含め、**すべてのプラグインに必要**です。
- ランタイムは引き続きプラグイン モジュールを個別にロードします。マニフェストは専用です
  発見+検証。
- 排他的なプラグインの種類は、`plugins.slots.*` を通じて選択されます。
  - `kind: "memory"` は `plugins.slots.memory` によって選択されます。
  - `kind: "context-engine"` は `plugins.slots.contextEngine` によって選択されます
    (デフォルト: 組み込み `legacy`)。
- プラグインがネイティブ モジュールに依存している場合は、ビルド手順と必要な手順を文書化します。
  パッケージマネージャーのホワイトリスト要件 (pnpm `allow-build-scripts` など)
  - `pnpm rebuild <package>`)。
