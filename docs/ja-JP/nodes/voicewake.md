# 音声ウェイク (グローバル ウェイク ワード)

OpenClaw は、**ウェイク ワードを**ゲートウェイ**が所有する単一のグローバル リスト**として扱います。

- **ノードごとのカスタム ウェイク ワード**はありません。
- **どのノード/アプリ UI でもリストを編集できます**。変更はゲートウェイによって保持され、全員にブロードキャストされます。
- macOS と iOS では、ローカル **Voice Wake の有効/無効** の切り替えが維持されます (ローカル UX + 権限は異なります)。
- Android は現在、Voice Wake をオフにしており、[Voice] タブで手動マイク フローを使用しています。

## ストレージ (ゲートウェイ ホスト)

ウェイク ワードは、ゲートウェイ マシンの次の場所に保存されます。

- `~/.openclaw/settings/voicewake.json`

形状:

```json
{ "triggers": ["openclaw", "claude", "computer"], "updatedAtMs": 1730000000000 }
```

## プロトコル

### メソッド

- `voicewake.get` → `{ triggers: string[] }`
- `voicewake.set` パラメータ付き `{ triggers: string[] }` → `{ triggers: string[] }`

注:

- トリガーは正規化されます (トリミングされ、空は削除されます)。空のリストはデフォルトに戻ります。
- 安全のために制限が適用されます (数/長さの上限)。

### イベント

- `voicewake.changed` ペイロード `{ triggers: string[] }`

受け取る人:

- すべての WebSocket クライアント (macOS アプリ、WebChat など)
- 接続されているすべてのノード (iOS/Android)、およびノード上でも初期の「現在の状態」プッシュとして接続されます。

## クライアントの動作

### macOS アプリ

- グローバル リストを使用して `VoiceWakeRuntime` トリガーを制御します。
- ボイス ウェイク設定で「トリガー ワード」を編集すると、`voicewake.set` が呼び出され、ブロードキャストに依存して他のクライアントとの同期が維持されます。

### iOS ノード- `VoiceWakeManager` トリガー検出にグローバル リストを使用します

- 設定でウェイク ワードを編集すると、`voicewake.set` (ゲートウェイ WS 経由) が呼び出され、ローカルのウェイク ワード検出の応答性も維持されます。

### Android ノード

- Voice Wake は現在、Android ランタイム/設定で無効になっています。
- Android の音声では、ウェイクワード トリガーの代わりに、[音声] タブの手動マイク キャプチャが使用されます。
