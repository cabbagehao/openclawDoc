# 画像とメディアのサポート — 2025-12-05

WhatsApp チャネルは **Baileys Web** 経由で実行されます。この文書には、送信、ゲートウェイ、およびエージェントの応答に関する現在のメディア処理ルールが記載されています。

## 目標

- `openclaw message send --media` 経由で、オプションのキャプションを含むメディアを送信します。
- Web 受信トレイからの自動返信に、テキストと一緒にメディアを含めることができます。
- タイプごとの制限を健全かつ予測可能に保ちます。

## CLI サーフェス

- `openclaw message send --media <path-or-url> [--message <caption>]`
  - `--media` オプション。メディアのみの送信の場合、キャプションは空にすることができます。
  - `--dry-run` は解決されたペイロードを出力します。 `--json` は `{ channel, to, messageId, mediaUrl, caption }` を生成します。

## WhatsApp Web チャネルの動作- 入力: ローカル ファイル パス **または** HTTP(S) URL

- フロー: バッファにロードし、メディアの種類を検出し、正しいペイロードを構築します。
  - **画像:** `agents.defaults.mediaMaxMb` (デフォルト 5MB) を対象として JPEG (最大横 2048px) にサイズ変更および再圧縮し、6MB を上限とします。
  - **オーディオ/音声/ビデオ:** 最大 16MB のパススルー。音声はボイスメモ (`ptt: true`) として送信されます。
  - **ドキュメント:** その他、最大 100MB、利用可能な場合はファイル名が保持されます。
- WhatsApp GIF スタイルの再生: `gifPlayback: true` (CLI: `--gif-playback`) を使用して MP4 を送信し、モバイル クライアントがインラインでループするようにします。
- MIME 検出では、マジック バイト、ヘッダー、ファイル拡張子の順に優先されます。
- キャプションは `--message` または `reply.text` から取得されます。空のキャプションは許可されます。
- ロギング: 非詳細は `↩️`/`✅` を示します。詳細には、サイズとソース パス/URL が含まれます。

## 自動応答パイプライン

- `getReplyFromConfig` は `{ text?, mediaUrl?, mediaUrls? }` を返します。
- メディアが存在する場合、Web 送信者は `openclaw message send` と同じパイプラインを使用してローカル パスまたは URL を解決します。
- 複数のメディア エントリが指定されている場合は、順次送信されます。

## コマンドへの受信メディア (Pi)- 受信 Web メッセージにメディアが含まれている場合、OpenClaw は一時ファイルにダウンロードし、テンプレート変数を公開します

- `{{MediaUrl}}` 受信メディアの疑似 URL。
- `{{MediaPath}}` コマンドを実行する前に書き込まれたローカル一時パス。
- セッションごとの Docker サンドボックスが有効になっている場合、受信メディアはサンドボックス ワークスペースにコピーされ、`MediaPath`/`MediaUrl` は `media/inbound/<filename>` のような相対パスに書き換えられます。
- メディア理解 (`tools.media.*` または共有 `tools.media.models` で構成されている場合) はテンプレート化の前に実行され、`[Image]`、`[Audio]`、および `[Video]` ブロックを `Body` に挿入できます。
  - オーディオは `{{Transcript}}` を設定し、コマンド解析にトランスクリプトを使用するため、スラッシュ コマンドは引き続き機能します。
  - ビデオと画像の説明では、コマンド解析のためにキャプション テキストが保存されます。
- デフォルトでは、最初に一致した画像/音声/ビデオ添付ファイルのみが処理されます。複数の添付ファイルを処理するには、`tools.media.<cap>.attachments` を設定します。

## 制限とエラー

**アウトバウンド送信の上限 (WhatsApp Web 送信)**

- 画像: 再圧縮後の最大 6MB の上限。
- オーディオ/音声/ビデオ: 上限 16MB。ドキュメント: 100MB の上限。
- サイズが大きすぎる、または読み取り不可能なメディア → ログのエラーがクリアされ、応答がスキップされます。

**メディア理解キャップ (転写/説明)**- 画像のデフォルト: 10MB (`tools.media.image.maxBytes`)。

- オーディオのデフォルト: 20MB (`tools.media.audio.maxBytes`)。
- ビデオのデフォルト: 50MB (`tools.media.video.maxBytes`)。
- 特大のメディアでは理解が省略されますが、返信は元の本文で引き続き行われます。

## テストのメモ

- 画像/音声/ドキュメントのケースの送信 + 返信フローをカバーします。
- 画像 (サイズ制限) の再圧縮とオーディオの音声メモ フラグを検証します。
- マルチメディア応答が順次送信としてファンアウトされるようにします。
