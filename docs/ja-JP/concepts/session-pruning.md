# セッションのプルーニング

セッション プルーニングは、各 LLM 呼び出しの直前に、メモリ内コンテキストから **古いツールの結果** をトリミングします。ディスク上のセッション履歴 (`*.jsonl`) は**書き換えられません**。

## 実行時

- `mode: "cache-ttl"` が有効で、セッションの最後の Anthropic 呼び出しが `ttl` より古い場合。
- そのリクエストのモデルに送信されるメッセージにのみ影響します。
- Anthropic API 呼び出し (および OpenRouter Anthropic モデル) に対してのみ有効です。
- 最良の結果を得るには、`ttl` をモデルの `cacheRetention` ポリシーに一致させます (`short` = 5 分、`long` = 1 時間)。
- プルーンの後、TTL ウィンドウがリセットされるため、`ttl` が再び期限切れになるまで、後続のリクエストはキャッシュを保持します。

## スマートなデフォルト (人為的)

- **OAuth または setup-token** プロファイル: `cache-ttl` プルーニングを有効にし、ハートビートを `1h` に設定します。
- **API キー** プロファイル: `cache-ttl` プルーニングを有効にし、ハートビートを `30m` に設定し、Anthropic モデルでデフォルトの `cacheRetention: "short"` を設定します。
- これらの値のいずれかを明示的に設定した場合、OpenClaw はそれらの値を**上書きしません**。

## これにより改善される内容 (コスト + キャッシュ動作)- **プルーニングの理由:** 人為的なプロンプト キャッシュは TTL 内にのみ適用されます。セッションが TTL を超えてアイドル状態になると、最初にプロンプ​​トをトリミングしない限り、次のリクエストで完全なプロンプトが再キャッシュされます

- **コストが安くなる:** プルーニングにより、TTL の有効期限が切れた後の最初のリクエストの **cacheWrite** サイズが削減されます。
- **TTL リセットが重要な理由:** プルーニングが実行されると、キャッシュ ウィンドウがリセットされるため、フォローアップ リクエストでは、履歴全体を再度キャッシュするのではなく、新しくキャッシュされたプロンプトを再利用できます。
- **機能しないこと:** プルーニングによってトークンが追加されたり、コストが「2 倍」になったりすることはありません。最初の post-TTL リクエストでキャッシュされる内容のみが変更されます。

## 剪定できるもの

- `toolResult` メッセージのみ。
- ユーザー + アシスタントのメッセージは**決して**変更されません。
- 最後の `keepLastAssistants` アシスタント メッセージは保護されます。そのカットオフ後のツールの結果は枝刈りされません。
- カットオフを確立するのに十分なアシスタント メッセージがない場合、プルーニングはスキップされます。
- **画像ブロック**を含むツール結果はスキップされます(トリミング/クリアされません)。

## コンテキストウィンドウの推定

プルーニングでは、推定されたコンテキスト ウィンドウ (文字数 ≈ トークン × 4) が使用されます。ベース ウィンドウは次の順序で解決されます。

1. `models.providers.*.models[].contextWindow` オーバーライド。
2. モデル定義 `contextWindow` (モデル レジストリから)。
3. デフォルトの `200000` トークン。

`agents.defaults.contextTokens` が設定されている場合、解決されたウィンドウの上限 (最小) として扱われます。

## モード

### キャッシュ-ttl- プルーニングは、最後の Anthropic 呼び出しが `ttl` (デフォルトは `5m`) より古い場合にのみ実行されます

- 実行時: 以前と同じソフトトリム + ハードクリア動作。

## ソフト剪定とハード剪定

- **ソフトトリム**: オーバーサイズのツール結果のみ。
  - 先頭と末尾を保持し、`...` を挿入し、元のサイズのメモを追加します。
  - 画像ブロックのある結果をスキップします。
- **ハードクリア**: ツールの結果全体を `hardClear.placeholder` に置き換えます。

## ツールの選択

- `tools.allow` / `tools.deny` は `*` ワイルドカードをサポートします。
- 否定の勝ち。
- 照合では大文字と小文字が区別されません。
- 空の許可リスト => すべてのツールが許可されます。

## 他の制限との相互作用

- 組み込みツールはすでに独自の出力を切り捨てています。セッション プルーニングは、長時間実行されるチャットによってモデル コンテキストにツール出力が過剰に蓄積されるのを防ぐ追加のレイヤーです。
- 圧縮は個別です。圧縮は要約して持続しますが、プルーニングはリクエストごとに一時的です。 [/concepts/compaction](/concepts/compaction) を参照してください。

## デフォルト (有効な場合)

- `ttl`: `"5m"`
- `keepLastAssistants`: `3`
- `softTrimRatio`: `0.3`
- `hardClearRatio`: `0.5`
- `minPrunableToolChars`: `50000`
- `softTrim`: `{ maxChars: 4000, headChars: 1500, tailChars: 1500 }`
- `hardClear`: `{ enabled: true, placeholder: "[Old tool result content cleared]" }`

## 例

デフォルト (オフ):

```json5
{
  agents: { defaults: { contextPruning: { mode: "off" } } },
}
```

TTL 対応のプルーニングを有効にします。

````json5
{
  agents: { defaults: { contextPruning: { mode: "cache-ttl", ttl: "5m" } } },
}
```プルーニングを特定のツールに制限します。

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        tools: { allow: ["exec", "read"], deny: ["*image*"] },
      },
    },
  },
}
````
