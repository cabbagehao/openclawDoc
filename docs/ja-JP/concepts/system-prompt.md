# システムプロンプト

OpenClaw は、エージェントの実行ごとにカスタム システム プロンプトを構築します。プロンプトは **OpenClaw が所有**しており、pi-coding-agent のデフォルト プロンプトを使用しません。

プロンプトは OpenClaw によって組み立てられ、各エージェントの実行に挿入されます。

## 構造

プロンプトは意図的にコンパクトになっており、固定セクションが使用されています。

- **ツール**: 現在のツールのリスト + 簡単な説明。
- **安全性**: 権力を求める行動や見落としを回避するための短いガードレールのリマインダー。
- **スキル** (利用可能な場合): オンデマンドでスキル命令をロードする方法をモデルに指示します。
- **OpenClaw 自己更新**: `config.apply` および `update.run` を実行する方法。
- **ワークスペース**: 作業ディレクトリ (`agents.defaults.workspace`)。
- **ドキュメント**: OpenClaw ドキュメントへのローカル パス (リポジトリまたは npm パッケージ) とそれらをいつ読むか。
- **ワークスペース ファイル (挿入)**: ブートストラップ ファイルが下に含まれていることを示します。
- **サンドボックス** (有効な場合): サンドボックス ランタイム、サンドボックス パス、および昇格された実行が利用可能かどうかを示します。
- **現在の日付と時刻**: ユーザーの現地時間、タイムゾーン、および時間形式。
- **返信タグ**: サポートされているプロバイダーのオプションの返信タグ構文。
- **ハートビート**: ハートビートのプロンプトと確認応答の動作。
- **実行時**: ホスト、OS、ノード、モデル、リポジトリ ルート (検出時)、思考レベル (1 行)。
- **Reasoning**: 現在の可視性レベル + /reasoning 切り替えヒント。システム プロンプトの安全ガードレールは推奨事項です。これらはモデルの動作をガイドしますが、ポリシーを強制するものではありません。ツールポリシー、実行承認、サンドボックス、およびチャネル許可リストを使用して、厳しい強制を行います。オペレータはこれらを仕様により無効にすることができます。

## 時間の処理

システム プロンプトには、専用の **現在の日付と時刻** セクションが含まれています。
ユーザーのタイムゾーンはわかっています。プロンプトのキャッシュを安定に保つために、現在は次の内容のみが含まれています。
**タイムゾーン** (動的な時計や時間形式はありません)。

エージェントが現在時刻を必要とする場合は、`session_status` を使用します。ステータスカード
タイムスタンプ行が含まれています。

次のように構成します。

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

完全な動作の詳細については、[日付と時刻](/date-time) を参照してください。

## スキル

適格なスキルが存在する場合、OpenClaw はコンパクトな **利用可能なスキル リスト**を挿入します
(`formatSkillsForPrompt`) これには、各スキルの **ファイル パス**が含まれます。の
プロンプトは、リストされた場所に `read` を使用して SKILL.md をロードするようにモデルに指示します。
場所 (ワークスペース、管理対象、またはバンドル)。資格のあるスキルがない場合は、
スキル欄は省略。

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

これにより、基本プロンプトが小さく保たれながら、ターゲットを絞ったスキルの使用が可能になります。
