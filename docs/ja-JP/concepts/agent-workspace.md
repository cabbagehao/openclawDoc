# エージェントワークスペース

ワークスペースはエージェントの自宅です。これは、次の目的で使用される唯一の作業ディレクトリです。
ファイル ツールとワークスペース コンテキスト用。プライベートに保ち、思い出として扱います。

これは、構成、認証情報、および
セッション。

**重要:** ワークスペースは **デフォルトの cwd** であり、ハード サンドボックスではありません。ツール
ワークスペースに対して相対パスを解決しますが、絶対パスでも到達できます。
サンドボックスが有効になっていない限り、ホスト上の他の場所にあります。隔離が必要な場合は、使用してください
[`agents.defaults.sandbox`](/gateway/sandboxing) (および/またはエージェントごとのサンドボックス構成)。
サンドボックスが有効で、`workspaceAccess` が `"rw"` ではない場合、ツールは動作します
ホスト ワークスペースではなく、`~/.openclaw/sandboxes` の下のサンドボックス ワークスペース内にあります。

## デフォルトの場所

- デフォルト: `~/.openclaw/workspace`
- `"default"` ではなく `OPENCLAW_PROFILE` が設定されている場合、デフォルトは次のようになります。
  `~/.openclaw/workspace-<profile>`。
- `~/.openclaw/openclaw.json` でオーバーライド:

```json5
{
  agent: {
    workspace: "~/.openclaw/workspace",
  },
}
```

`openclaw onboard`、`openclaw configure`、または `openclaw setup` は、
ワークスペースに保存し、ブートストラップ ファイルが存在しない場合はシードします。
サンドボックス シード コピーは、通常のワークスペース内のファイルのみを受け入れます。シンボリックリンク/ハードリンク
ソース ワークスペースの外で解決されるエイリアスは無視されます。

すでにワークスペース ファイルを自分で管理している場合は、ブートストラップを無効にすることができます
ファイルの作成:

```json5
{ agent: { skipBootstrap: true } }
```

## 追加のワークスペース フォルダー古いインストールでは `~/openclaw` が作成されている可能性があります。複数のワークスペースを維持する

周囲にディレクトリがあると、認証や状態のドリフトが混乱する可能性があります。
ワークスペースは一度にアクティブになります。

**推奨事項:** 単一のアクティブなワークスペースを維持します。使用しなくなった場合は、
余分なフォルダーを削除するには、アーカイブするか、ゴミ箱に移動します (例: `trash ~/openclaw`)。
意図的に複数のワークスペースを保持する場合は、次のことを確認してください。
`agents.defaults.workspace` はアクティブなものを指します。

`openclaw doctor` は、余分なワークスペース ディレクトリを検出すると警告します。

## ワークスペース ファイル マップ (各ファイルの意味)

OpenClaw がワークスペース内で期待する標準ファイルは次のとおりです。

- `AGENTS.md`
  - エージェントの操作手順とメモリの使用方法。
  - すべてのセッションの開始時にロードされます。
  - ルール、優先順位、「どのように行動するか」の詳細を記載するのに適しています。

- `SOUL.md`
  - ペルソナ、トーン、境界線。
  - セッションごとにロードされます。

- `USER.md`
  - ユーザーが誰であるか、およびユーザーにどのように対処するか。
  - セッションごとにロードされます。

- `IDENTITY.md`
  - エージェントの名前、雰囲気、絵文字。
  - ブートストラップ儀式中に作成/更新されます。

- `TOOLS.md`
  - ローカル ツールと規約に関するメモ。
  - ツールの可用性を制御しません。それは単なる指導です。

- `HEARTBEAT.md`
  - ハートビート実行用のオプションの小さなチェックリスト。
  - トークンのバーンを避けるために短くしてください。- `BOOT.md`
  - 内部フックが有効になっている場合、ゲートウェイの再起動時に実行されるオプションの起動チェックリスト。
  - 短くしてください。アウトバウンド送信にはメッセージ ツールを使用します。

- `BOOTSTRAP.md`
  - 1 回限りの初回実行の儀式。
  - 新しいワークスペースに対してのみ作成されます。
  - 儀式が完了したら削除してください。

- `memory/YYYY-MM-DD.md`
  - 日次メモリログ (1 日あたり 1 ファイル)。
  - セッション開始時に今日と昨日を読むことをお勧めします。

- `MEMORY.md` (オプション)
  - 厳選された長期記憶。
  - メインのプライベート セッションでのみロードします (共有/グループ コンテキストではありません)。

ワークフローと自動メモリフラッシュについては、[メモリ](/concepts/memory) を参照してください。

- `skills/` (オプション)
  - ワークスペース固有のスキル。
  - 名前が衝突した場合、管理/バンドルされたスキルをオーバーライドします。

- `canvas/` (オプション)
  - ノード表示用のキャンバス UI ファイル (`canvas/index.html` など)。

ブートストラップ ファイルが欠落している場合、OpenClaw は「欠落ファイル」マーカーを
セッションが終了し、続行します。大きなブートストラップ ファイルは、挿入時に切り詰められます。
`agents.defaults.bootstrapMaxChars` (デフォルト: 20000) で制限を調整し、
`agents.defaults.bootstrapTotalMaxChars` (デフォルト: 150000)。
`openclaw setup` は、既存のものを上書きせずに、欠落しているデフォルトを再作成できます。
ファイル。

## ワークスペースにないもの

これらは `~/.openclaw/` の下に存在するため、ワークスペース リポジトリにコミットしないでください。- `~/.openclaw/openclaw.json` (構成)

- `~/.openclaw/credentials/` (OAuth トークン、API キー)
- `~/.openclaw/agents/<agentId>/sessions/` (セッショントランスクリプト + メタデータ)
- `~/.openclaw/skills/` (管理スキル)

セッションまたは構成を移行する必要がある場合は、それらを個別にコピーして保管してください。
バージョン管理外。

## Git バックアップ (推奨、プライベート)

ワークスペースをプライベート メモリとして扱います。 **プライベート** git リポジトリに入れてください。
バックアップされており、回復可能です。

ゲートウェイが実行されているマシン (つまり、
ワークスペースは存続します)。

### 1) リポジトリを初期化する

git がインストールされている場合、新しいワークスペースは自動的に初期化されます。これなら
ワークスペースがまだリポジトリではない場合は、次を実行します。

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md SOUL.md TOOLS.md IDENTITY.md USER.md HEARTBEAT.md memory/
git commit -m "Add agent workspace"
```

### 2) プライベート リモートを追加します (初心者向けのオプション)

オプション A: GitHub Web UI

1. GitHub 上に新しい **プライベート** リポジトリを作成します。
2. README を使用して初期化しないでください (マージ競合を回避します)。
3. HTTPS リモート URL をコピーします。
4. リモコンを追加して、次を押します。

```bash
git branch -M main
git remote add origin <https-url>
git push -u origin main
```

オプション B: GitHub CLI (`gh`)

```bash
gh auth login
gh repo create openclaw-workspace --private --source . --remote origin --push
```

オプション C: GitLab Web UI

1. GitLab 上に新しい **プライベート** リポジトリを作成します。
2. README を使用して初期化しないでください (マージ競合を回避します)。
3. HTTPS リモート URL をコピーします。
4. リモコンを追加して、次を押します。

```bash
git branch -M main
git remote add origin <https-url>
git push -u origin main
```

### 3) 継続的なアップデート

```bash
git status
git add .
git commit -m "Update memory"
git push
```

## シークレットをコミットしないでください

プライベート リポジトリであっても、ワークスペースにシークレットを保存しないでください。- API キー、OAuth トークン、パスワード、またはプライベート認証情報。

- `~/.openclaw/` の下にあるもの。
- チャットまたは機密添付ファイルの生のダンプ。

機密参照を保存する必要がある場合は、プレースホルダーを使用して実際の参照を保存してください。
シークレットは他の場所 (パスワード マネージャー、環境変数、または `~/.openclaw/`) にあります。

推奨される `.gitignore` スターター:

```gitignore
.DS_Store
.env
**/*.key
**/*.pem
**/secrets*
```

## ワークスペースを新しいマシンに移動する

1. リポジトリのクローンを目的のパス (デフォルト `~/.openclaw/workspace`) に作成します。
2. `agents.defaults.workspace` を `~/.openclaw/openclaw.json` のそのパスに設定します。
3. `openclaw setup --workspace <path>` を実行して、不足しているファイルをシードします。
4. セッションが必要な場合は、`~/.openclaw/agents/<agentId>/sessions/` を
   古いマシンは別途。

## 高度なメモ

- マルチエージェント ルーティングでは、エージェントごとに異なるワークスペースを使用できます。参照
  ルーティング設定用の [チャネル ルーティング](/channels/channel-routing)。
- `agents.defaults.sandbox` が有効な場合、メイン以外のセッションはセッションごとのサンドボックスを使用できます
  `agents.defaults.sandbox.workspaceRoot` の下のワークスペース。
