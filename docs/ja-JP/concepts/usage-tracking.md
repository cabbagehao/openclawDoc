# 使用状況の追跡

## それは何ですか

- プロバイダーの使用量/割り当てを使用量エンドポイントから直接取得します。
- 推定コストはありません。プロバイダーによって報告されたウィンドウのみ。

## それが現れる場所

- チャット内の `/status`: セッション トークンと推定コストを含む絵文字が豊富なステータス カード (API キーのみ)。プロバイダーの使用状況は、**現在のモデルのプロバイダー** (利用可能な場合) について表示されます。
- チャットの `/usage off|tokens|full`: 応答ごとの使用状況フッター (OAuth ではトークンのみが表示されます)。
- チャット内の `/usage cost`: OpenClaw セッション ログから集計されたローカル コストの概要。
- CLI: `openclaw status --usage` はプロバイダーごとの完全な内訳を出力します。
- CLI: `openclaw channels list` は、プロバイダー設定と一緒に同じ使用状況のスナップショットを出力します (スキップするには `--no-usage` を使用します)。
- macOS メニュー バー: コンテキストの下の「使用法」セクション (利用可能な場合のみ)。

## プロバイダー + 認証情報

- **Anthropic (Claude)**: 認証プロファイル内の OAuth トークン。
- **GitHub Copilot**: 認証プロファイル内の OAuth トークン。
- **Gemini CLI**: 認証プロファイル内の OAuth トークン。
- **反重力**: 認証プロファイル内の OAuth トークン。
- **OpenAI Codex**: 認証プロファイル内の OAuth トークン (存在する場合は accountId が使用されます)。
- **MiniMax**: API キー (コーディング プラン キー; `MINIMAX_CODE_PLAN_KEY` または `MINIMAX_API_KEY`); 5 時間のコーディング プラン ウィンドウを使用します。
- **z.ai**: env/config/auth ストア経由の API キー。

一致する OAuth/API 認証情報が存在しない場合、使用状況は非表示になります。
