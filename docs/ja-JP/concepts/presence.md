# プレゼンス

OpenClaw の「プレゼンス」は、以下の軽量かつベストエフォート型のビューです。

- **ゲートウェイ**自体、および
- **ゲートウェイに接続されているクライアント** (Mac アプリ、WebChat、CLI など)

プレゼンスは主に、macOS アプリの **インスタンス** タブをレンダリングするために使用されます。
オペレーターに迅速な可視性を提供します。

## プレゼンスフィールド (表示される内容)

プレゼンス エントリは、次のようなフィールドを持つ構造化オブジェクトです。

- `instanceId` (オプションですが強く推奨): 安定したクライアント ID (通常は `connect.client.instanceId`)
- `host`: 人間に優しいホスト名
- `ip`: ベストエフォート IP アドレス
- `version`: クライアントのバージョン文字列
- `deviceFamily` / `modelIdentifier`: ハードウェアのヒント
- `mode`: `ui`、`webchat`、`cli`、`backend`、`probe`、`test`、 `node`、...
- `lastInputSeconds`: 「最後のユーザー入力からの秒数」(既知の場合)
- `reason`: `self`、`connect`、`node-connected`、`periodic`、...
- `ts`: 最終更新タイムスタンプ (エポックからのミリ秒)

## プロデューサー (存在感が生まれる場所)

プレゼンス エントリは複数のソースによって生成され、**マージ**されます。

### 1) ゲートウェイのセルフエントリー

ゲートウェイは起動時に常に「self」エントリをシードするため、UI にはゲートウェイ ホストが表示されます。
クライアントが接続する前であっても。

### 2) WebSocket 接続すべての WS クライアントは `connect` リクエストで始まります。ハンドシェイクが成功すると、

ゲートウェイは、その接続のプレゼンス エントリを更新/挿入します。

#### 1 回限りの CLI コマンドが表示されない理由

CLI は多くの場合、短い 1 回限りのコマンドのために接続します。スパム送信を避けるため、
インスタンス リスト、`client.mode === "cli"` はプレゼンス エントリに**されません**。

### 3) `system-event` ビーコン

クライアントは、`system-event` メソッドを介して、より豊富な定期ビーコンを送信できます。マック
アプリはこれを使用して、ホスト名、IP、および `lastInputSeconds` を報告します。

### 4) ノード接続 (役割: ノード)

ノードがゲートウェイ WebSocket 経由で `role: node` を使用して接続すると、ゲートウェイ
そのノードのプレゼンス エントリを更新/挿入します (他の WS クライアントと同じフロー)。

## マージ + 重複排除ルール (`instanceId` が重要な理由)

プレゼンス エントリは、単一のメモリ内マップに保存されます。

- エントリは **プレゼンス キー**によってキー設定されます。
- 最適なキーは、再起動後も存続する安定した `instanceId` (`connect.client.instanceId` から) です。
- キーでは大文字と小文字が区別されません。

クライアントが安定した `instanceId` なしで再接続すると、
**重複**行。

## TTL と制限されたサイズ

存在は意図的に一時的です:

- **TTL:** 5 分より古いエントリは削除されます
- **最大エントリ:** 200 (古いものから順に削除)

これにより、リストが常に最新の状態に保たれ、無制限のメモリ増加が回避されます。

## リモート/トンネルに関する注意事項 (ループバック IP)クライアントが SSH トンネル/ローカル ポート転送経由で接続すると、ゲートウェイは

リモート アドレスは `127.0.0.1` として参照してください。クライアントから報告された適切な情報の上書きを避けるため
IP、ループバック リモート アドレスは無視されます。

## 消費者

### [macOS インスタンス] タブ

macOS アプリは `system-presence` の出力をレンダリングし、小さなステータスを適用します
最終更新の経過時間に基づいたインジケーター (アクティブ/アイドル/古い)。

## デバッグのヒント

- 生のリストを表示するには、ゲートウェイに対して `system-presence` を呼び出します。
- 重複が見つかった場合:
  - クライアントがハンドシェイクで安定した `client.instanceId` を送信することを確認します
  - 定期的なビーコンが同じ `instanceId` を使用していることを確認します
  - 接続派生エントリが欠落しているかどうかを確認します `instanceId` (重複が予想されます)
