# コンテキストウィンドウと圧縮

すべてのモデルには **コンテキスト ウィンドウ** (表示できる最大トークン) があります。長時間実行されるチャットはメッセージとツールの結果を蓄積します。制限が厳しくなると、OpenClaw は制限内に収まるように古い履歴を **圧縮**します。

## 圧縮とは

圧縮 **古い会話をコンパクトな概要エントリに要約**し、最近のメッセージをそのまま保持します。概要はセッション履歴に保存されるため、今後のリクエストでは以下が使用されます。

- 圧縮の概要
- 圧縮ポイント以降の最近のメッセージ

圧縮はセッションの JSONL 履歴に**残ります**。

## 構成

`openclaw.json` の `agents.defaults.compaction` 設定を使用して、圧縮動作 (モード、ターゲット トークンなど) を構成します。
圧縮要約では、デフォルトで不透明な識別子が保持されます (`identifierPolicy: "strict"`)。これを `identifierPolicy: "off"` でオーバーライドすることも、`identifierPolicy: "custom"` および `identifierInstructions` でカスタム テキストを指定することもできます。

必要に応じて、`agents.defaults.compaction.model` を介して圧縮要約に別のモデルを指定できます。これは、プライマリ モデルがローカル モデルまたは小規模モデルであり、より有能なモデルによって生成された圧縮サマリーが必要な場合に便利です。オーバーライドは、任意の `provider/model-id` 文字列を受け入れます。

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "openrouter/anthropic/claude-sonnet-4-5"
      }
    }
  }
}
```

これは、ローカル モデル、たとえば、要約専用の 2 番目の Ollama モデルや、微調整された圧縮スペシャリストでも機能します。

````json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "ollama/llama3.1:8b"
      }
    }
  }
}
```設定を解除すると、圧縮ではエージェントのプライマリ モデルが使用されます。

## 自動圧縮 (デフォルトはオン)

セッションがモデルのコンテキスト ウィンドウに近づくか超えると、OpenClaw は自動圧縮をトリガーし、圧縮されたコンテキストを使用して元のリクエストを再試行する場合があります。

次のことがわかります:

- `🧹 Auto-compaction complete` 冗長モード
- `/status` は `🧹 Compactions: <count>` を表示しています

圧縮する前に、OpenClaw は **サイレント メモリ フラッシュ**を実行して保存することができます。
永続的なメモをディスクに保存します。詳細と構成については、[メモリ](/concepts/memory) を参照してください。

## 手動圧縮

`/compact` (オプションで指示付き) を使用して圧縮パスを強制します。

````

/compact Focus on decisions and open questions

```

## コンテキストウィンドウのソース

コンテキスト ウィンドウはモデル固有です。 OpenClaw は、構成されたプロバイダー カタログからのモデル定義を使用して制限を決定します。

## 圧縮と枝刈り

- **圧縮**: JSONL に要約して**永続化**します。
- **セッション プルーニング**: 古い **ツール結果**のみを、**メモリ内**でリクエストごとにトリミングします。

プルーニングの詳細については、[/concepts/session-pruning](/concepts/session-pruning) を参照してください。

## OpenAI サーバー側の圧縮

OpenClaw は、OpenAI Responses のサーバー側圧縮ヒントもサポートしています。
互換性のある直接 OpenAI モデル。これはローカルの OpenClaw とは別のものです
圧縮し、並行して実行できます。- ローカル圧縮: OpenClaw は要約してセッション JSONL に保持します。
- サーバー側の圧縮: OpenAI は、次の場合にプロバイダー側でコンテキストを圧縮します。
  `store` + `context_management` が有効になります。

モデルのパラメータとオーバーライドについては、[OpenAI プロバイダー](/providers/openai) を参照してください。

## ヒント

- セッションが古く感じられる場合、またはコンテキストが肥大化している場合は、`/compact` を使用します。
- 大きなツール出力はすでに切り詰められています。枝刈りを行うと、ツールによる結果の蓄積をさらに減らすことができます。
- 新しいスレートが必要な場合は、`/new` または `/reset` によって新しいセッション ID が開始されます。
```
