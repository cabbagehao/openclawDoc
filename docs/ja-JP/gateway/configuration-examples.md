# 構成例

以下の例は、現在の構成スキーマに合わせてあります。包括的なリファレンスとフィールドごとのメモについては、[構成](/gateway/configuration) を参照してください。

## クイックスタート

### 絶対最小値

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

`~/.openclaw/openclaw.json` に保存すると、その番号からボットに DM を送信できます。

### 推奨スターター

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-5" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## よくあるパターン

### マルチプラットフォームのセットアップ

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### セキュア DM モード (共有受信箱 / マルチユーザー DM)

複数の人がボットに DM できる場合 (`allowFrom` の複数のエントリ、複数人の承認のペアリング、または `dmPolicy: "open"`)、**セキュア DM モード** を有効にして、異なる送信者からの DM がデフォルトで 1 つのコンテキストを共有しないようにします。

```json5
{
  // Secure DM mode (recommended for multi-user or sensitive DM agents)
  session: { dmScope: "per-channel-peer" },

  channels: {
    // Example: WhatsApp multi-user inbox
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // Example: Discord multi-user inbox
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```

Discord/Slack/Google Chat/MS Teams/Mattermost/IRC の場合、送信者の認証はデフォルトで ID が優先されます。
そのリスクを明示的に受け入れる場合にのみ、各チャネルの `dangerouslyAllowNameMatching: true` との直接変更可能な名前/電子メール/ニックネームの一致を有効にしてください。

### Anthropic セットアップ トークン + API キー、MiniMax フォールバック

<Warning>

Claude Code 以外での Anthropic セットアップ トークンの使用は、一部の環境で制限されています
過去のユーザー。これをユーザー選択のリスクとして扱い、現在の Anthropic を検証します
サブスクリプション認証に応じて、以前の条件が適用されます。
</Warning>

```json5

  auth: {
    profiles: {
      "anthropic:subscription": {
        provider: "anthropic",

      anthropic: ["anthropic:subscription", "anthropic:api"],

      primary: "anthropic/claude-opus-4-6",

```

## ヒント

- `dmPolicy: "open"` を設定する場合、一致する `allowFrom` リストには `"*"` が含まれている必要があります。
- プロバイダー ID が異なります (電話番号、ユーザー ID、チャネル ID)。プロバイダーのドキュメントを使用して形式を確認してください。
- 後で追加するオプションのセクション: `web`、`browser`、`ui`、`discovery`、`canvasHost`、`talk`、`signal`、 `imessage`。
- セットアップに関する詳細な注意事項については、[プロバイダー](/providers) および [トラブルシューティング](/gateway/troubleshooting) を参照してください。
