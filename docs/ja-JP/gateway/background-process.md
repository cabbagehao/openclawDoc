# バックグラウンド実行 + プロセスツール

OpenClaw は、`exec` ツールを通じてシェル コマンドを実行し、長時間実行されるタスクをメモリ内に保持します。 `process` ツールは、これらのバックグラウンド セッションを管理します。

## 実行ツール

主要なパラメータ:

- `command` (必須)
- `yieldMs` (デフォルト 10000): この遅延後の自動バックグラウンド
- `background` (ブール値): すぐにバックグラウンドにします
- `timeout` (秒、デフォルトは 1800): このタイムアウト後にプロセスを強制終了します。
- `elevated` (ブール値): 昇格モードが有効/許可されている場合、ホスト上で実行されます。
- 本物の TTY が必要ですか? `pty: true` を設定します。
- `workdir`、`env`

動作:

- フォアグラウンド実行は出力を直接返します。
- バックグラウンド (明示的またはタイムアウト) の場合、ツールは `status: "running"` + `sessionId` と短いテールを返します。
- セッションがポーリングされるかクリアされるまで、出力はメモリに保持されます。
- `process` ツールが禁止されている場合、`exec` は同期的に実行され、`yieldMs`/`background` は無視されます。
- 生成された実行コマンドは、コンテキスト認識シェル/プロファイル ルールの `OPENCLAW_SHELL=exec` を受け取ります。

## 子プロセスのブリッジング実行/プロセス ツールの外部で長時間実行される子プロセス (CLI リスポーンやゲートウェイ ヘルパーなど) を生成する場合は、終了信号が転送され、終了/エラー時にリスナーが切り離されるように、子プロセス ブリッジ ヘルパーをアタッチします。これにより、systemd 上の孤立したプロセスが回避され、プラットフォーム間で一貫したシャットダウン動作が維持されます

環境の上書き:

- `PI_BASH_YIELD_MS`: デフォルトの収量 (ミリ秒)
- `PI_BASH_MAX_OUTPUT_CHARS`: メモリ内出力上限 (文字数)
- `OPENCLAW_BASH_PENDING_MAX_OUTPUT_CHARS`: ストリームごとの保留中の標準出力/標準エラー出力の上限 (文字数)
- `PI_BASH_JOB_TTL_MS`: 終了したセッションの TTL (ミリ秒、1 分～3 時間の範囲)

構成 (推奨):

- `tools.exec.backgroundMs` (デフォルトは 10000)
- `tools.exec.timeoutSec` (デフォルトは 1800)
- `tools.exec.cleanupMs` (デフォルトは 1800000)
- `tools.exec.notifyOnExit` (デフォルトは true): バックグラウンド実行が終了したときに、システム イベントと要求ハートビートをキューに入れます。
- `tools.exec.notifyOnExitEmptySuccess` (デフォルト false): true の場合、出力を生成しなかった成功したバックグラウンド実行の完了イベントもキューに入れます。

## プロセスツール

アクション:- `list`: 実行中 + 終了したセッション

- `poll`: セッションの新しい出力をドレインします (終了ステータスもレポートします)。
- `log`: 集約された出力を読み取ります (`offset` + `limit` をサポート)
- `write`: 標準入力 (`data`、オプションの `eof`) を送信します。
- `kill`: バックグラウンド セッションを終了します
- `clear`: 終了したセッションをメモリから削除します
- `remove`: 実行中の場合は強制終了、それ以外の場合は終了した場合はクリア

注:

- バックグラウンドセッションのみがリストされ、メモリ内に保持されます。
- プロセスの再起動時にセッションが失われます (ディスクの永続性はありません)。
- セッション ログは、`process poll/log` を実行し、ツールの結果が記録された場合にのみチャット履歴に保存されます。
- `process` はエージェントごとにスコープが定められています。そのエージェントによって開始されたセッションのみが表示されます。
- `process list` には、クイック スキャン用の派生 `name` (コマンド動詞 + ターゲット) が含まれています。
- `process log` は、ラインベースの `offset`/`limit` を使用します。
- `offset` と `limit` の両方を省略すると、最後の 200 行が返され、ページング ヒントが含まれます。
  ・ `offset` を指定し、`limit` を省略した場合は、`offset` から最後まで戻ります (200 に制限されません)。

## 例

長いタスクを実行し、後でポーリングします。

```json
{ "tool": "exec", "command": "sleep 5 && echo done", "yieldMs": 1000 }
```

```json
{ "tool": "process", "action": "poll", "sessionId": "<id>" }
```

バックグラウンドですぐに開始します。

```json
{ "tool": "exec", "command": "npm run build", "background": true }
```

標準入力を送信します:

```json
{ "tool": "process", "action": "write", "sessionId": "<id>", "data": "y\n" }
```
