# ブリッジプロトコル (レガシーノードトランスポート)

Bridge プロトコルは **レガシー** ノード トランスポート (TCP JSONL) です。新しいノードクライアント
代わりに、統合ゲートウェイ WebSocket プロトコルを使用する必要があります。

オペレーターまたはノードクライアントを構築している場合は、
[ゲートウェイ プロトコル](/gateway/protocol)。

**注意:** 現在の OpenClaw ビルドには TCP ブリッジ リスナーが同梱されていません。この文書は歴史的参照のために保管されています。
レガシー `bridge.*` 構成キーは構成スキーマの一部ではなくなりました。

## 両方ある理由

- **セキュリティ境界**: ブリッジは、
  完全なゲートウェイ API サーフェス。
- **ペアリング + ノード ID**: ノードの許可はゲートウェイによって所有され、関連付けられます。
  ノードごとのトークンに変換します。
- **検出 UX**: ノードは LAN 上の Bonjour 経由でゲートウェイを検出したり、接続したりできます。
  テールネットの真上。
- **ループバック WS**: SSH 経由でトンネルされない限り、完全な WS コントロール プレーンはローカルのままです。

## 輸送

- TCP、1 行に 1 つの JSON オブジェクト (JSONL)。
- オプションの TLS (`bridge.tls.enabled` が true の場合)。
- 従来のデフォルトのリスナー ポートは `18790` でした (現在のビルドでは TCP ブリッジが開始されません)。TLS が有効な場合、検出 TXT レコードには `bridgeTls=1` に加えて
  `bridgeTlsSha256` は秘密ではないヒントです。 Bonjour/mDNS TXT レコードは
  認証されていない;クライアントは、アドバタイズされたフィンガープリントを
  明示的なユーザーの意図やその他の帯域外検証を行わない権限のある PIN。

## ハンドシェイク + ペアリング

1. クライアントは、ノードのメタデータ + トークン (すでにペアになっている場合) を含む `hello` を送信します。
2. ペアになっていない場合、ゲートウェイは `error` (`NOT_PAIRED`/`UNAUTHORIZED`) を応答します。
3. クライアントは `pair-request` を送信します。
4. ゲートウェイは承認を待ってから、`pair-ok` と `hello-ok` を送信します。

`hello-ok` は `serverName` を返し、`canvasHostUrl` を含む場合があります。

## フレーム

クライアント → ゲートウェイ:

- `req` / `res`: スコープ付きゲートウェイ RPC (チャット、セッション、構成、ヘルス、ボイスウェイク、スキル.bins)
- `event`: ノード信号 (音声トランスクリプト、エージェント要求、チャット購読、実行ライフサイクル)

ゲートウェイ → クライアント:

- `invoke` / `invoke-res`: ノード コマンド (`canvas.*`、`camera.*`、`screen.record`、
  `location.get`、`sms.send`)
- `event`: 購読済みセッションのチャット更新
- `ping` / `pong`: キープアライブ

従来の許可リストの適用は `src/gateway/server-bridge.ts` に存在していました (削除されました)。

## ライフサイクル イベントの実行ノードは、`exec.finished` または `exec.denied` イベントを発行して、system.run アクティビティを表示できます

これらはゲートウェイ内のシステム イベントにマッピングされます。 (レガシー ノードは依然として `exec.started` を発行する可能性があります。)

ペイロード フィールド (注記がない限り、すべてオプション):

- `sessionKey` (必須): システム イベントを受信するエージェント セッション。
- `runId`: グループ化用の一意の実行 ID。
- `command`: 未加工またはフォーマットされたコマンド文字列。
- `exitCode`、`timedOut`、`success`、`output`: 完了の詳細 (完了のみ)。
- `reason`: 拒否理由 (拒否のみ)。

## テールネットの使用法

- ブリッジをテールネット IP にバインドします: `bridge.bind: "tailnet"`
  `~/.openclaw/openclaw.json`。
- クライアントは、MagicDNS 名またはテールネット IP を介して接続します。
- Bonjour はネットワークを**越えません**。手動のホスト/ポートまたは広域 DNS‑SD を使用する
  必要なときに。

## バージョン管理

ブリッジは現在 **暗黙的 v1** (最小/最大ネゴシエーションなし) です。下位互換性
期待されています。重大な変更を行う前に、ブリッジ プロトコルのバージョン フィールドを追加します。
