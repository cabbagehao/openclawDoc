# ドクター

`openclaw doctor` は、OpenClaw の修復および移行ツールです。古くなったものを修正します
config/state を確認し、健全性をチェックし、実行可能な修復手順を提供します。

## クイックスタート

```bash
openclaw doctor
```

### ヘッドレス / 自動化

```bash
openclaw doctor --yes
```

プロンプトを表示せずにデフォルトを受け入れます (該当する場合、再起動/サービス/サンドボックス修復手順を含む)。

```bash
openclaw doctor --repair
```

プロンプトを表示せずに推奨される修復を適用します (修復 + 安全な場所で再起動)。

```bash
openclaw doctor --repair --force
```

積極的な修復も適用します（カスタム スーパーバイザ設定を上書きします）。

```bash
openclaw doctor --non-interactive
```

プロンプトを表示せずに実行し、安全な移行 (構成の正規化 + ディスク上の状態の移動) のみを適用します。人間による確認が必要な再起動/サービス/サンドボックスアクションをスキップします。
レガシー状態の移行は、検出されると自動的に実行されます。

```bash
openclaw doctor --deep
```

追加のゲートウェイ インストールのシステム サービスをスキャンします (launchd/systemd/schtasks)。

書き込む前に変更を確認したい場合は、まず設定ファイルを開きます。

```bash
cat ~/.openclaw/openclaw.json
```

## 機能 (概要)- git インストール用のオプションのプレフライト アップデート (対話型のみ)

- UI プロトコルの鮮度チェック (プロトコル スキーマが新しい場合にコントロール UI を再構築します)。
- ヘルスチェック + 再起動プロンプト。
- スキルステータスの概要 (適格/欠落/ブロック)。
- レガシー値の正規化を構成します。
- OpenCode Zen プロバイダーのオーバーライド警告 (`models.providers.opencode`)。
- 従来のディスク上の状態の移行 (セッション/エージェント ディレクトリ/WhatsApp 認証)。
- レガシー cron ストアの移行 (`jobId`、`schedule.cron`、トップレベルの配信/ペイロード フィールド、ペイロード `provider`、単純な `notify: true` Webhook フォールバック ジョブ)。
- 状態の整合性と権限のチェック (セッション、トランスクリプト、状態ディレクトリ)。
- ローカルで実行する場合の構成ファイルのアクセス許可チェック (chmod 600)。
- モデル認証の健全性: OAuth の有効期限をチェックし、期限切れのトークンを更新し、認証プロファイルのクールダウン/無効状態を報告します。
- 余分なワークスペース ディレクトリの検出 (`~/openclaw`)。
- サンドボックスが有効な場合のサンドボックス イメージの修復。
- レガシー サービスの移行と追加のゲートウェイの検出。
- ゲートウェイのランタイム チェック (サービスはインストールされているが実行されていない、キャッシュされた launchd ラベル)。
- チャネル ステータスの警告 (実行中のゲートウェイから調査)。
- スーパーバイザ設定監査 (launchd/systemd/schtasks) (オプションで修復あり)。
- ゲートウェイ ランタイムのベスト プラクティス チェック (ノードと Bun、バージョン マネージャー パス)。
- ゲートウェイ ポートの衝突診断 (デフォルト `18789`)。- オープン DM ポリシーのセキュリティ警告。
- ゲートウェイ認証はローカル トークン モードをチェックします (トークン ソースが存在しない場合にトークン生成を提供します。トークン SecretRef 構成は上書きされません)。
- Linux 上の systemd リンガー チェック。
- ソースのインストール チェック (pnpm ワークスペースの不一致、UI アセットの欠落、tsx バイナリの欠落)。
- 更新された構成 + ウィザードのメタデータを書き込みます。

### 2) 従来の構成キーの移行

構成に非推奨のキーが含まれている場合、他のコマンドは実行を拒否し、質問します。
`openclaw doctor` を実行してください。

医師は次のことを行います:

- 適用された移行を表示します。
- 更新されたスキーマで `~/.openclaw/openclaw.json` を書き換えます。

また、ゲートウェイは、起動時に医師の移行を自動実行します。
従来の設定形式であるため、古い設定は手動介入なしで修復されます。

現在の移行:- `routing.allowFrom` → `channels.whatsapp.allowFrom`

- `routing.groupChat.requireMention` → `channels.whatsapp/telegram/imessage.groups."*".requireMention`
- `routing.groupChat.historyLimit` → `messages.groupChat.historyLimit`
- `routing.groupChat.mentionPatterns` → `messages.groupChat.mentionPatterns`
- `routing.queue` → `messages.queue`
- `routing.bindings` → トップレベル `bindings`
- `routing.agents`/`routing.defaultAgentId` → `agents.list` + `agents.list[].default`
- `routing.agentToAgent` → `tools.agentToAgent`
- `routing.transcribeAudio` → `tools.media.audio.models`
- `bindings[].match.accountID` → `bindings[].match.accountId`
- `accounts` という名前が付いているが、`accounts.default` が欠落しているチャネルの場合、アカウント スコープのトップレベルの単一アカウント チャネル値が存在する場合は `channels.<channel>.accounts.default` に移動します。
- `identity` → `agents.list[].identity`
- `agent.*` → `agents.defaults` + `tools.*` (ツール/昇格/実行/サンドボックス/サブエージェント)
- `agent.model`/`allowedModels`/`modelAliases`/`modelFallbacks`/`imageModelFallbacks`
  → `agents.defaults.models` + `agents.defaults.model.primary/fallbacks` + `agents.defaults.imageModel.primary/fallbacks`
- `browser.ssrfPolicy.allowPrivateNetwork` → `browser.ssrfPolicy.dangerouslyAllowPrivateNetwork`

医師の警告には、マルチアカウント チャネルに対するアカウントのデフォルトに関するガイダンスも含まれています。

- 2 つ以上の `channels.<channel>.accounts` エントリが `channels.<channel>.defaultAccount` または `accounts.default` なしで構成されている場合、医師は、フォールバック ルーティングによって予期しないアカウントが選択される可能性があると警告します。
- `channels.<channel>.defaultAccount` が不明なアカウント ID に設定されている場合、医師は警告し、設定されているアカウント ID をリストします。

### 2b) OpenCode Zen プロバイダーのオーバーライド`models.providers.opencode` (または `opencode-zen`) を手動で追加した場合、

`@mariozechner/pi-ai` の組み込み OpenCode Zen カタログをオーバーライドします。それはできる
すべてのモデルを単一の API に強制するか、コストをゼロにします。医師はそうするように警告します
オーバーライドを削除し、モデルごとの API ルーティングとコストを復元します。

### 3) レガシー状態の移行 (ディスク レイアウト)

Doctor は、古いディスク上のレイアウトを現在の構造に移行できます。

- セッション ストア + トランスクリプト:
  - `~/.openclaw/sessions/` から `~/.openclaw/agents/<agentId>/sessions/`
- エージェントのディレクトリ:
  - `~/.openclaw/agent/` から `~/.openclaw/agents/<agentId>/agent/`
- WhatsApp 認証状態 (ベイリーズ):
  - 従来の `~/.openclaw/credentials/*.json` から (`oauth.json` を除く)
  - `~/.openclaw/credentials/whatsapp/<accountId>/...` へ (デフォルトのアカウント ID: `default`)

これらの移行はベストエフォート型で冪等です。医師は次の場合に警告を発します。
従来のフォルダーはバックアップとして残されます。ゲートウェイ/CLI も自動移行します
起動時のレガシーセッション + エージェントディレクトリなので、履歴/認証/モデルは
手動によるドクターランの必要のないエージェントごとのパス。 WhatsApp認証は意図的にのみ行われます
`openclaw doctor` 経由で移行されました。

### 3b) 従来の cron ストアの移行

Doctor は cron ジョブ ストア (デフォルトでは `~/.openclaw/cron/jobs.json`) もチェックします。
またはオーバーライドされた場合は `cron.store`) スケジューラがまだ使用している古いジョブ シェイプの場合
互換性のために受け入れます。

現在の cron クリーンアップには次のものが含まれます。- `jobId` → `id`

- `schedule.cron` → `schedule.expr`
- 最上位ペイロード フィールド (`message`、`model`、`thinking`、...) → `payload`
- 最上位の配信フィールド (`deliver`、`channel`、`to`、`provider`、...) → `delivery`
- ペイロード `provider` 配信エイリアス → 明示的 `delivery.channel`
- 単純な従来の `notify: true` Webhook フォールバック ジョブ → 明示的な `delivery.mode="webhook"` と `delivery.to=cron.webhook`

Doctor は、`notify: true` ジョブを自動的に移行できる場合にのみ自動移行します。
行動を変えること。ジョブが従来の通知フォールバックと既存の通知フォールバックを組み合わせた場合
非 Webhook 配信モードの場合、医師は警告し、そのジョブを手動レビューのために残します。

### 4) 状態の整合性チェック (セッションの永続性、ルーティング、安全性)

状態ディレクトリは操作上の脳幹です。消えたら負け
セッション、資格情報、ログ、構成 (他の場所にバックアップがある場合を除く)。

医師は次のことをチェックします。- **状態ディレクトリがありません**: 壊滅的な状態損失について警告し、再作成するように求めます
ディレクトリを削除し、欠落したデータを回復できないことを通知します。

- **状態ディレクトリ権限**: 書き込み可能性を検証します。権限を修復することを提案する
  (所有者/グループの不一致が検出されると、`chown` ヒントが発行されます)。
- **macOS クラウド同期状態ディレクトリ**: iCloud Drive で状態が解決されると警告します
  (`~/Library/Mobile Documents/com~apple~CloudDocs/...`) または
  `~/Library/CloudStorage/...` 同期バックされたパスにより I/O が遅くなる可能性があるため
  レースのロック/同期も可能です。
- **Linux SD または eMMC 状態ディレクトリ**: 状態が `mmcblk*` に解決されると警告します
  SD または eMMC ベースのランダム I/O が遅くなり、消耗する可能性があるため、ソースをマウントします。
  セッションと資格情報の書き込みが高速になります。
- **セッション ディレクトリがありません**: `sessions/` とセッション ストア ディレクトリは次のとおりです。
  履歴を保持し、`ENOENT` のクラッシュを回避するために必要です。
- **トランスクリプトの不一致**: 最近のセッション エントリが欠落している場合に警告します
  トランスクリプトファイル。
- **メイン セッション「1 行 JSONL」**: メイン トランスクリプトに 1 つしかない場合にフラグを立てます。
  行（履歴は蓄積されていません）。
- **複数の状態ディレクトリ**: 複数の `~/.openclaw` フォルダーが存在する場合に警告します
  ホーム ディレクトリ、または `OPENCLAW_STATE_DIR` が他の場所を指している場合 (履歴は
  インストール間で分割します)。
- **リモート モード リマインダー**: `gateway.mode=remote` の場合、医師はあなたに走るようリマインドします。
  それはリモートホスト上にあります (状態はそこに存在します)。- **構成ファイルのアクセス許可**: `~/.openclaw/openclaw.json` が次の場合に警告します。
  グループ/ワールドで読み取り可能であり、`600` に厳格化することを提案しています。

### 5) モデル認証の健全性 (OAuth の有効期限)

Doctor は認証ストア内の OAuth プロファイルを検査し、トークンが不正な場合に警告します。
期限切れ/有効期限が切れており、安全なときに更新できます。人間のクロードコードなら
プロファイルが古い場合は、`claude setup-token` を実行する (またはセットアップ トークンを貼り付ける) ことを提案します。
更新プロンプトは対話型 (TTY) で実行している場合にのみ表示されます。 `--non-interactive`
更新試行をスキップします。

Doctor は、次の理由で一時的に使用できない認証プロファイルも報告します。

- 短いクールダウン (レート制限/タイムアウト/認証失敗)
- 無効化が長くなる (請求/クレジットの失敗)

### 12) ゲートウェイ認証チェック (ローカル トークン)

Doctor は、ローカル ゲートウェイ トークン認証の準備状況をチェックします。

- トークン モードにトークンが必要で、トークン ソースが存在しない場合、ドクターはトークンの生成を提案します。
- `gateway.auth.token` が SecretRef で管理されているが利用できない場合、医師は警告し、平文で上書きしません。
- `openclaw doctor --generate-gateway-token` は、トークン SecretRef が構成されていない場合にのみ生成を強制します。

### 12b) 読み取り専用 SecretRef 対応修復

一部の修復フローでは、ランタイムのフェールファスト動作を弱めることなく、構成された資格情報を検査する必要があります。- `openclaw doctor --fix` は、ターゲット設定修復のための status ファミリ コマンドと同じ読み取り専用 SecretRef 概要モデルを使用するようになりました。

- 例: Telegram `allowFrom` / `groupAllowFrom` `@username` 修復は、利用可能な場合、構成されたボット資格情報の使用を試みます。
- Telegram ボット トークンが SecretRef 経由で設定されているが、現在のコマンド パスで利用できない場合、ドクターは、資格情報は設定されているが利用できないと報告し、トークンがクラッシュしたり、トークンが見つからないと誤って報告したりするのではなく、自動解決をスキップします。

### 15) スーパーバイザ設定の監査と修復

Doctor は、インストールされているスーパーバイザ設定 (launchd/systemd/schtasks) をチェックします。
欠落している、または古いデフォルト (systemd のネットワークとオンラインの依存関係など)
再起動の遅延）。不一致が見つかった場合は、アップデートを推奨し、
サービス ファイル/タスクを現在のデフォルトに書き換えます。

注:- スーパーバイザ設定を書き換える前に、`openclaw doctor` プロンプトが表示されます。

- `openclaw doctor --yes` は、デフォルトの修復プロンプトを受け入れます。
- `openclaw doctor --repair` は、プロンプトを表示せずに推奨される修正を適用します。
- `openclaw doctor --repair --force` はカスタム スーパーバイザ設定を上書きします。
- トークン認証にトークンが必要で、`gateway.auth.token` が SecretRef で管理されている場合、ドクター サービスのインストール/修復は SecretRef を検証しますが、解決されたプレーンテキストのトークン値をスーパーバイザ サービス環境のメタデータに保持しません。
- トークン認証にトークンが必要で、構成されたトークン SecretRef が未解決の場合、ドクターは実用的なガイダンスを示してインストール/修復パスをブロックします。
- `gateway.auth.token` と `gateway.auth.password` の両方が構成され、`gateway.auth.mode` が設定されていない場合、モードが明示的に設定されるまで、ドクターはインストール/修復をブロックします。
- Linux user-systemd ユニットの場合、サービス認証メタデータを比較するときに、ドクター トークン ドリフト チェックに `Environment=` ソースと `EnvironmentFile=` ソースの両方が含まれるようになりました。
- `openclaw gateway install --force` 経由でいつでも完全な書き換えを強制できます。

### 16) ゲートウェイ ランタイム + ポート診断

Doctor はサービス ランタイム (PID、最後の終了ステータス) を検査し、サービス ランタイムが終了した場合に警告します。
サービスはインストールされていますが、実際には実行されていません。ポートの衝突もチェックします
ゲートウェイ ポート (デフォルト `18789`) を調べ、考えられる原因をレポートします (ゲートウェイはすでに
実行中、SSH トンネル)。

### 17) ゲートウェイ ランタイムのベスト プラクティスゲートウェイ サービスが Bun またはバージョン管理されたノード パスで実行されると Doctor が警告する

(`nvm`、`fnm`、`volta`、`asdf` など)。 WhatsApp + Telegram チャネルにはノードが必要です。
サービスが機能しないため、アップグレード後にバージョン マネージャー パスが壊れる可能性があります。
シェルの初期化をロードします。医師は、次の場合にシステム ノード インストールへの移行を提案します。
利用可能 (Homebrew/apt/choco)。

Doctor は設定変更を永続化し、ウィザードのメタデータをスタンプして記録します。
ドクターラン。

### 19) ワークスペースのヒント (バックアップ + メモリ システム)
