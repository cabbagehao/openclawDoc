# スクリプト

`scripts/` ディレクトリには、ローカル ワークフローと運用タスク用のヘルパー スクリプトが含まれています。
これらは、タスクがスクリプトに明確に関連付けられている場合に使用します。それ以外の場合は、CLI を使用することをお勧めします。

## 規約

- ドキュメントまたはリリース チェックリストで参照されていない限り、スクリプトは**オプション**です。
- CLI サーフェスが存在する場合は、CLI サーフェスを優先します (例: 認証監視では `openclaw models status --check` を使用します)。
- スクリプトはホスト固有であると想定します。新しいマシンで実行する前に読んでください。

## 認証監視スクリプト

認証監視スクリプトは次の場所に文書化されています。
[/automation/auth-monitoring](/automation/auth-monitoring)

## スクリプトを追加する場合

- スクリプトに焦点を当てて文書化してください。
- 関連するドキュメントに短いエントリを追加します (不足している場合は作成します)。
