# 統合ランタイムストリーミングリファクタリング計画

## 目的

`main`、`subagent`、および `acp` に 1 つの共有ストリーミング パイプラインを提供することで、すべてのランタイムが同一の結合、チャンク化、配信順序、およびクラッシュ回復動作を取得します。

## これが存在する理由

- 現在の動作は、複数のランタイム固有のシェーピング パスに分割されています。
- フォーマット/結合のバグは 1 つのパスでは修正できますが、他のパスでは依然として残ります。
- 配信の一貫性、重複の抑制、および回復のセマンティクスを推論するのが難しくなります。

## ターゲット アーキテクチャ

単一パイプライン、ランタイム固有のアダプター:

1. ランタイムアダプターは正規イベントのみを発行します。
2. 共有ストリーム アセンブラが結合し、テキスト/ツール/ステータス イベントを終了します。
3. 共有チャンネル プロジェクターは、チャンネル固有のチャンキング/フォーマットを 1 回適用します。
4. 共有配信台帳により、冪等の送信/再生セマンティクスが強制されます。
5. アウトバウンドチャネルアダプタは送信を実行し、配信チェックポイントを記録します。

正規イベント契約:

- `turn_started`
- `text_delta`
- `block_final`
- `tool_started`
- `tool_finished`
- `status`
- `turn_completed`
- `turn_failed`
- `turn_cancelled`

## ワークストリーム

### 1) 正規ストリーミング契約- 厳密なイベント スキーマ + 検証をコアで定義します

- 各ランタイムが互換性のあるイベントを発行することを保証するアダプター コントラクト テストを追加します。
- 不正なランタイム イベントを早期に拒否し、構造化された診断を表面化します。

### 2) 共有ストリームプロセッサ

- ランタイム固有のコアレッサー/プロジェクター ロジックを 1 つのプロセッサーに置き換えます。
- プロセッサは、テキスト デルタ バッファリング、アイドル フラッシュ、最大チャンク分割、および完了フラッシュを所有します。
- ドリフトを防ぐために、ACP/メイン/サブエージェント構成解決を 1 つのヘルパーに移動します。

### 3) 共有チャネルの投影

- チャネル アダプターをダムに保つ: 完了したブロックを受け入れて送信します。
- Discord 固有のチャンキングの癖をチャンネル プロジェクターのみに移動します。
- 投影前にパイプラインのチャネルに依存しないようにします。

### 4) 納品台帳 + リプレイ

- ターンごと/チャンクごとの配信 ID を追加します。
- 物理的な送信の前後にチェックポイントを記録します。
- 再起動時に、保留中のチャンクを冪等に再生し、重複を避けます。

### 5) 移行とカットオーバー

- フェーズ 1: シャドウ モード (新しいパイプラインは出力を計算しますが、古いパスは送信します。比較)。
- フェーズ 2: ランタイムごとのカットオーバー (`acp`、次に `subagent`、その後 `main`、またはリスクによる逆転)。
- フェーズ 3: 従来のランタイム固有のストリーミング コードを削除します。

## 非目標- このリファクタリングでは、ACP ポリシー/権限モデルへの変更はありません

- 投影互換性修正以外のチャネル固有の機能拡張はありません。
- トランスポート/バックエンドの再設計はありません (イベント パリティに必要な場合を除き、acpx プラグイン コントラクトはそのまま残ります)。

## リスクと緩和策

- リスク: 既存のメイン/サブエージェント パスにおける動作の退行。
  軽減策: シャドウ モードの差分 + アダプター コントラクト テスト + チャネル e2e テスト。
- リスク: クラッシュ回復中に送信が重複する。
  軽減策: 耐久性のある配信 ID + 配信アダプターでの冪等な再生。
- リスク: ランタイム アダプターが再び分岐します。
  軽減策: すべてのアダプターに共有コントラクト テスト スイートが必要です。

## 受け入れ基準

- すべてのランタイムは共有ストリーミング コントラクト テストに合格します。
- Discord ACP/メイン/サブエージェントは、小さなデルタに対して同等の間隔/チャンキング動作を生成します。
- クラッシュ/リスタートリプレイでは、同じ配信 ID の重複したチャンクは送信されません。
- レガシー ACP プロジェクター/コアレッサー パスが削除されました。
- ストリーミング構成の解決は共有され、ランタイムに依存しません。
