# モデル構成 (探索)

このドキュメントには、将来のモデル構成に関する**アイデア**が記載されています。それはではありません
出荷仕様。現在の動作については、以下を参照してください。

- [モデル](/concepts/models)
- [モデルフェイルオーバー](/concepts/model-failover)
- [OAuth + プロファイル](/concepts/oauth)

## モチベーション

オペレーターは次のことを望んでいます。

- プロバイダーごとに複数の認証プロファイル (個人用と仕事用)。
- 予測可能なフォールバックを備えたシンプルな `/model` 選択。
- テキスト モデルと画像対応モデルを明確に分離します。

## 可能な方向 (高レベル)

- モデルの選択をシンプルにします: `provider/model` とオプションのエイリアス。
- プロバイダーに明示的な順序で複数の認証プロファイルを持たせます。
- すべてのセッションが一貫してフェイルオーバーできるように、グローバル フォールバック リストを使用します。
- 明示的に構成されている場合のみ、イメージ ルーティングをオーバーライドします。
