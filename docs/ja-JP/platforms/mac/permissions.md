# macOS 権限 (TCC)

macOS の権限付与は脆弱です。 TCC は許可付与を
アプリのコード署名、バンドル識別子、およびディスク上のパス。それらのいずれかが変化した場合、
macOS はアプリを新しいものとして扱い、プロンプトを削除または非表示にする場合があります。

## 安定した権限の要件

- 同じパス: 固定の場所からアプリを実行します (OpenClaw の場合、`dist/OpenClaw.app`)。
- 同じバンドル ID: バンドル ID を変更すると、新しい権限 ID が作成されます。
- 署名付きアプリ: 署名されていないビルドまたはアドホック署名されたビルドは権限を保持しません。
- 一貫した署名: 本物の Apple 開発証明書または開発者 ID 証明書を使用します。
  そのため、署名は再構築しても安定したままになります。

アドホック署名は、ビルドごとに新しい ID を生成します。 macOS は以前のものを忘れます
を許可すると、古いエントリがクリアされるまでプロンプトが完全に表示されなくなる場合があります。

## プロンプトが表示されなくなった場合の回復チェックリスト

1. アプリを終了します。
2. [システム設定] -> [プライバシーとセキュリティ] でアプリのエントリを削除します。
3. 同じパスからアプリを再起動し、権限を再付与します。
4. それでもプロンプトが表示されない場合は、`tccutil` を使用して TCC エントリをリセットし、再試行します。
5. 一部の権限は、macOS を完全に再起動した後にのみ再表示されます。

リセットの例 (必要に応じてバンドル ID を置き換えます):

```bash
sudo tccutil reset Accessibility ai.openclaw.mac
sudo tccutil reset ScreenCapture ai.openclaw.mac
sudo tccutil reset AppleEvents
```

## ファイルとフォルダーのアクセス許可 (デスクトップ/ドキュメント/ダウンロード)macOS は、ターミナル/バックグラウンド プロセスのデスクトップ、ドキュメント、ダウンロードをゲートすることもあります。ファイルの読み取りやディレクトリのリストがハングする場合は、ファイル操作を実行するのと同じプロセス コンテキスト (たとえば、ターミナル/iTerm、LaunchAgent 起動アプリ、SSH プロセス) へのアクセスを許可します

回避策: フォルダーごとの許可を回避したい場合は、ファイルを OpenClaw ワークスペース (`~/.openclaw/workspace`) に移動します。

アクセス許可をテストする場合は、必ず実際の証明書を使用して署名してください。アドホック
ビルドは、アクセス許可が重要ではない、ローカルでの簡単な実行にのみ受け入れられます。
