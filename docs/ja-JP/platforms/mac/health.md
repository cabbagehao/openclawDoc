# macOS でのヘルスチェック

メニューバーアプリからリンクされたチャンネルが健全かどうかを確認する方法。

## メニューバー

- ステータスドットがベイリーの健康状態を反映するようになりました。
  - 緑: リンク済み + ソケットが最近オープンされました。
  - オレンジ色: 接続/再試行中。
  - 赤: ログアウトしたか、プローブが失敗しました。
- 2 番目の行には「linked · auth 12m」と表示されるか、失敗の理由が表示されます。
- 「ヘルスチェックの実行」メニュー項目は、オンデマンドプローブをトリガーします。

## 設定

- [全般] タブには、リンクされた認証経過時間、セッション ストアのパス/カウント、最終チェック時間、最終エラー/ステータス コード、およびヘルス チェックの実行/ログの表示のボタンを示すヘルス カードが表示されます。
- キャッシュされたスナップショットを使用するため、UI は即座に読み込まれ、オフライン時に正常にフォールバックします。
- **[チャネル]タブ**には、チャネルのステータスとWhatsApp/Telegramのコントロール(ログインQR、ログアウト、プローブ、最後の切断/エラー)が表示されます。

## プローブの仕組み

- アプリは `ShellExecutor` 経由で `openclaw health --json` を約 60 秒ごとにオンデマンドで実行します。プローブは、メッセージを送信せずに認証情報をロードし、ステータスを報告します。
- ちらつきを避けるために、最後の良好なスナップショットと最後のエラーを個別にキャッシュします。それぞれのタイムスタンプを表示します。

## 迷ったときは

- [ゲートウェイの健全性](/gateway/health) (`openclaw status`、`openclaw status --deep`、`openclaw health --json`) および `web-heartbeat` の末尾 `/tmp/openclaw/openclaw-*.log` の CLI フローは引き続き使用できます。 `web-reconnect`。
